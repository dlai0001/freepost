Command line

Freepost ships a headless CLI — the Newman analog — that runs your collections with the exact same engine as the desktop app. Same .curl files, same variables, same pm.* scripts. Perfect for CI.

Install

Build the standalone bundle from a checkout and run it with Node (v20+): 

npm install
npm run build:cli
node out/cli/index.mjs run ./my-collection

The bin entry is freepost, so an npm link (or installing the package) lets you call freepost directly.

freepost run <collection>

Runs every one-shot request in the collection, sharing one session, and exits non-zero if anything fails.

FlagMeaning
--env <file>Environment JSON file (session > env > request params).
--workflow <path>Run a workflow instead of loose requests. Repeatable.
--filter <substr>Only run requests whose path contains the substring. Repeatable.
--bailStop at the first failing request or step.
--reporter <cli|json>Human output (default) or a JSON array of reports for machines.

Exit codes: 0 everything passed · 1 a request errored, a test failed, or a workflow halted · 2 a usage error (bad flag, missing collection).

What runs, what's skipped

One-shot requests run: HTTP (.curl), gRPC unary (.grpc), and MQTT publish (.mqtt). Long-lived requests are skipped with a note at the end of the run: WebSocket (.ws) and MQTT subscribe. (gRPC server-streaming can't be told apart without loading the proto, so it runs and reports a clear "use the streaming client" error.) authorization_code OAuth needs a token cached by a prior interactive sign-in — see Authentication.

freepost mock <collection> [--port <n>]

Starts the mock server from the collection's saved examples and serves until Ctrl-C. Omit --port for an OS-assigned port. Great for standing up fake dependencies in local dev or integration jobs. 

freepost mock ./my-collection --port 8080

In CI (GitHub Actions)

A typical job that fails the build when any request or assertion fails:

- name: API smoke tests
  run: |
    npm ci
    npm run build:cli
    node out/cli/index.mjs run ./collection \
      --env ci.env.json --bail --reporter json

Because collections are plain files in your repo, they're versioned and reviewed like any other code — no exported artifact, no account, nothing to sync.
