Installing Freepost

Download the installer for your platform, or build from source. Every release is published straight from main by GitHub Actions — the same source you can audit and build yourself.

Download

PlatformFile
macOS (Intel & Apple Silicon)freepost-mac.dmg
Windows (64-bit)freepost-setup.exe
Linux (portable)freepost.AppImage
Linux (Debian/Ubuntu)freepost.deb

These links always resolve to the newest release. Browse all releases for older versions.

Opening an unsigned build

Freepost's binaries are not code-signed — buying signing certificates would mean either a paid tier or a corporate identity, and there will never be either. The source is public and the builds are reproducible, so you can trust the app the same way you'd trust any dependency: by reading it. Because it's unsigned, your OS will show a one-time warning the first time you open it.

macOS

Open the .dmg and drag Freepost to Applications. The first time you launch it, macOS Gatekeeper says the app "cannot be opened because the developer cannot be verified."

  1. Right-click (or Control-click) the Freepost app and choose Open.
  2. Click Open in the dialog that appears.

You only need to do this once. If the app is reported as "damaged," clear the quarantine flag in Terminal: 

xattr -dr com.apple.quarantine /Applications/Freepost.app

Windows

Run freepost-setup.exe. Windows SmartScreen may show a blue "Windows protected your PC" screen for an unrecognized publisher.

  1. Click More info.
  2. Click Run anyway.

Linux

The AppImage is self-contained — make it executable and run it:

chmod +x freepost.AppImage
./freepost.AppImage

Or install the Debian package:

sudo dpkg -i freepost.deb

Build from source instead

If you'd rather not run a downloaded binary at all, build your own from a checkout (Node v20+): 

git clone https://github.com/dlai0001/freepost
cd freepost
npm install
npm start

That runs the desktop app in dev mode. To produce an installer for your own platform, use npm run build:mac, build:win, or build:linux.
