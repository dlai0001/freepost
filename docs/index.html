The API client that

An open-source Postman alternative for REST, GraphQL, WebSocket, gRPC, MQTT, and MCP. No account. No cloud. Zero network calls except the requests you send. Your collections are plain curl files on disk — runnable, diffable, yours.

Your requests are just curl

#!/usr/bin/env bash
# ---
# description: Fetches a single user record by id
# label: [users, smoke]
# ---

BASE_URL="${BASE_URL:-https://api.example.com}"
TOKEN="${TOKEN:?}"

curl --request GET \
  --url "https://${BASE_URL}/api/users/42" \
  --header 'Accept: application/json' \
  --header "Authorization: Bearer ${TOKEN}"

Every saved request is a valid shell script. bash runs it. git diff reviews it. Any tool that imports curl understands it.

Built for developers behind firewalls

Truly offline

No registration, no login, no sync, no telemetry, no update pings. The only module that can open a socket is the request engine — and CI enforces it.

Files, anywhere

Collections are folders you choose. Filenames are request names. Version them with git, review them in pull requests, carry them on a USB stick.

Postman-compatible scripting

Pre-request and test scripts with the pm.* API and Chai assertions. Import your existing Postman collections and keep your tests.

Import OpenAPI & Swagger

Point Freepost at any OpenAPI/Swagger spec URL, preview every endpoint, and pick exactly which ones to bring in. Each becomes a runnable curl file with sensible placeholder values already filled in.

OAuth2, including interactive sign-in

client_credentials, password, and the browser-based authorization_code flow — PKCE by default, the token cached under .freepost/ and refreshed automatically. The sign-in opens your system browser and catches the redirect on a local loopback port.

Mock server from your examples

Serve your saved response examples over a local HTTP port — no real backend needed. Mark an example active, or force one with a query param. Unmatched paths return 404 and never proxy out, so it stays fully offline. Runs from the app or freepost mock in CI.

gRPC, the curl way

Call unary and server-streaming gRPC methods with a grpcurl-style .grpc file. Bring your .proto files, edit the request as JSON, and read the reply (or a live stream) — schema stays local, so it works fully offline.

MQTT publish & subscribe

Talk to MQTT brokers with mosquitto-style .mqtt files. Publish a payload with QoS and retain, or subscribe to a topic (wildcards included) and watch messages stream in live. TLS, auth, and ${VAR} all supported.

MCP servers, testable

Call an MCP server's tools, resources, and prompts over stdio or Streamable HTTP from a .mcp file — a real MCP Inspector invocation. Then assert on the result with pm.* tests and run them in CI, which no other MCP client does. freepost mcp check turns silent tool-schema drift into a failing build, and a stdio server never spawns until you've seen the exact command and approved it.

Workflows

Chain requests into ordered runs. Mark steps that are expected to fail. Capture a token in step one, use it in step three.

Search & labels

Label requests in frontmatter and find anything instantly — by name, label, description, URL, or method.

REST, GraphQL, WebSocket

GraphQL queries stored readable, not JSON-escaped. WebSocket sessions saved as websocat commands with message presets.
