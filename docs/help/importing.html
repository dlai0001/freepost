Importing

Open a collection, then click Import in the top bar. The Import dialog has three modes, each writing one or more .curl files straight into your open collection.

Import from curl (paste)

Switch to the Paste command tab and paste a curl, websocat, or wscat command copied from a terminal, browser devtools "Copy as cURL", or another API tool. Give it a name, or leave it blank and Freepost derives one from the URL. The import is lenient — unsupported flags are kept as a note rather than dropped silently. 

curl -X POST https://api.example.com/users \
  -H 'Content-Type: application/json' \
  -d '{"name": "ada"}'

Import a file

Switch to the File tab and browse to either:

  • A Postman collection (v2.1 JSON export) — folders, requests, and pm.* scripts import as-is; Postman's {{var}} syntax converts to Freepost's ${VAR}.
  • Any shell script containing a single curl, websocat, or wscat command.

Import from an OpenAPI/Swagger URL

Switch to the OpenAPI/Swagger URL tab, paste the spec URL (OpenAPI 3.x or Swagger 2.0, JSON or YAML), and click Fetch. Endpoints are listed grouped by tag, each with a checkbox — select individually, per-group, or Select all. Optionally set a destination folder to import everything under one prefix instead of each operation's own tag/path folder.

Each fetched operation is grouped by tag/folder with a tri-state select-all checkbox at every level.

Re-importing is safe to repeat
Imports only ever add new .curl files — nothing else in your collection is touched. Run an OpenAPI import again later and just select the new endpoints.
